Computer preparation

For this course, we will use R for instruction. Of course, you can use any language you want for your own projects or for this course, but the course materials slides and case studies use R, and we can best assist you if you are using R.

Our goal in this session is to make sure everyone has the R statistical computing environment running on their computer, and to do some basic simulations for ODE models.

Install R

Go to http://www.r-project.org and click download R.

It may ask you to choose a CRAN mirror (i.e.an R distribution site). Scroll down to USA and click any of the links associated with a nearby institution.

Under download and install R, choose the option that is relevant for your operating system.

Follow the instructions to install R.

Working with R

You will interact with R mostly via the console, or command line.

You can also write R scripts, which are text documents containing R code that you can ask R to run all at once.

The basics

There are two main R screens we may be working in: the command line (aka R console) and R documents (aka R scripts).

The command line is what you see when you first open R. When you input code here and hit return, R executes the command you have given it. For example, try writing

2 + 2
## [1] 4

R Documents

You can open an R document by going to File -> New Document (on a Mac) or File -> New Script (on Windows). In an R document, you can write and edit multiple lines of code. R will not run this code until you copy and paste it into the command line. You can save R documents and open them again later.

In this course, we will frequently give you R files to run as they are. To do this, save the file in the folder you want. Next open R and change your working directory to that folder. Then go to File -> Open Document (on a Mac) or File -> Open Script (on a Windows) and choose the file you want to run. Select all of the document, copy it, paste it into the command window, and hit return. Your code should run successfully!

Working Directory

Create a folder/directory on your computer for this course. For example /Users/forrest/Documents/phmodeling on a Mac, or C:\Users\Forrest\Documents.

Your working directory is the default file that R uses when you want to open files, read in files, or save files. The first thing you should do when opening R is to change your working directory to the folder you want to work in. On a Mac, you do this by going to Misc -> Change Working Directory. On a Windows computer, you do this by going to File -> Change Dir.

Change your working directory to the folder you created for this course.

Basics

If you already know how to use R, you can skip this section and go directly to the modeling part below.

If you do not have experience with R, the following brief tutorial outlines some of the basic features of the R language, its syntax, and built-in functions.

Variable assignment

Variable assignment in R is easy, and variable types do not generally need to be defined in advance. To assign the value 7 to the variable x, you can write

x = 7
x <- 7

The <- assignment operator is mostly like =. We will primarily use = in this course.

Vectors

You can create vectors using the concatenate function

Vectors can have named elements

You can create a vector consisting of a sequence of numbers using the : operator

1:10

Matrices

You can create a matrix as follows

m = matrix(1:9, nrow = 3, ncol = 3)

Note that the first argument to the matrix function is a vector of numbers; these are filled into the matrix column-wise unless the byrow=TRUE argument is specified.

Matrix/vector arithmetic

y = 7:9

# element-wise multiplication
x * y
## [1]  7 16 27
# dot product
x %*% y
##      [,1]
## [1,]   50
A = matrix(1:9, nrow = 3)


# this is matrix-vector multiplication
A %*% x
##      [,1]
## [1,]   30
## [2,]   36
## [3,]   42

R does some strange things when you ask it to multiply objects with incompatible dimensions.

# this does NOT give an error

## [1,]    1    4    7
## [2,]    4   10   16
## [3,]    9   18   27

In this example, element-wise left-multiplication of a matrix by a vector gives column-wise multiplication. Beware!

Instead of a numeric index to a vector or matrix, you can use logical indexing via a binary operation that returns TRUE or FALSE.

Functions

To make a function in R, use the following syntax

myfun = function(x) {
    y = x * x
    return(y)
}

Now you can call this function as follows:

myfun(7)
## [1] 49

Function arguments can have default values,

myfun2 = function(x = 2) {
    y = x * x
    return(y)
}

Now you can call myfun2 with no arguments, and the default argument x=2 will be used.

myfun2()
## [1] 4

To learn more about any built-in (or package) function, type

`?`(function_name)

and you will see the manual page for function_name.

The sample function is an easy way to draw a random value from a set of values.

x = 1:10
p = runif(10)

# draw one value from x uniformly at random
sample(x, 1)
## [1] 3
# draw one value from x with probability proportional to its
# element of `p`

Beware: if x is a scalar, sample will draw instead from the intgegers 1:floor(x).

By default, if the number of samples desired is greater than 1, sample will draw values without replacement. You can use the argument replace=TRUE to draw samples larger than the number of elements in x.

Running a simple SI model with vital dynamics

Consider a simple deterministic Susceptible-Infective model with individuals born into the susceptible category with rate \(\lambda\), and dying with rates \(\mu_s\) for susceptibles and \(\mu_i\) for infectives. We can write this ODE system as follows.

\[ \begin{aligned} \frac{dS}{dt} = \lambda - \beta S(t) I(t) - \mu_s S(t) \\ \frac{dI}{dt} = \beta S(t) I(t) - \mu_i I(t) \end{aligned} \]

Make sure you understand the dyanmics implied by this system.

When you are ready, run the following code in R. Make sure you understand the relationship between the equations above and the SI_model function below.

library(deSolve)

######################## SI Model ########################

SI_model <- function(times, yinit, pars) {
    with(as.list(c(yinit, pars)), {
        dS <- lam - beta * I * S - mu_s * S
        dI <- beta * I * S - mu_i * I
        return(list(c(dS, dI)))
    })
}

################## Setting Parameters ##################

beta = 8e-06  # beta is transmission parameter
mu_s = 1/30  # mu_s is the mortality rate for susceptibles
mu_i = 1/10  # mu_i is the mortality rate for infecteds
lam = 1000  # lam is the birth rate (assumed not to depend on population size)
pars <- cbind(lam, beta, mu_s, mu_i)

############### Setting Time Frame ####################

start_time = 0  # start date
end_time = 200  # end date 
times <- seq(start_time, end_time, by = 1)  # gives a sequence from start to end

############ Setting Initial Conditions ###############

yinit <- c(S = 3000, I = 10)

############# Running The Model ####################

results <- as.data.frame(ode(y = yinit, times = times, func = SI_model, 
    parms = pars, method = "rk4"))

############# Plotting Results ######################

plot(results$time, results$S, col = "green", type = "l", bty = "n", 
    lwd = 2, xlab = "Time", ylab = "Number", ylim = range(c(results$S, 
        results$I)))
lines(results$time, results$I, col = "red", lwd = 2)
legend(100, 20000, c("Susceptible", "Infective"), col = c("green", 
    "red"), lwd = 2)
